Термины
* - важное уточнение
Логин и Пароль - учетные данные, используемые для входа в систему. Логин идентифицирует пользователя, а пароль защищает его данные.
Номер акта - уникальный идентификатор записи в базе данных, представляющий собой целое число.
Срочность - поле, указывающее на срочность ремонта инструмента. 0 - несрочный ремонт (40 дней), 1 - срочный ремонт (15 дней).
Запись - строка с данными о конкретном клиенте.
БД - База данных
Бэкап - это создание резервных копий файлов или документов

Инструкция по работе с программой
1. При запуске Вас встретит окно Входа, где нужно в соответствующие поля ввести Логин и Пароль (Логин: 1 Пароль:1) при первом входе, при последующих - пропуск
2. Отображается основная БД со всей информацией по клиентам. Далее если хотите добавить запись -> п.3, если хотите удалить запись -> п.2.1
2.1 Нажимаем левым щелчком мыши (далее - просто нажимаем) на ячейку с Номером акта, который хочется удалить
2.2 Нажимаем на кнопку "Удалить" - запись удаляется из БД
3. Нажимаем на кнопку "Добавить" - в БД добавляется строка, где автоматически вставляются поля "Номер акта" (автоинкремент) и "Дата принятия" (текущая дата)
4. Заполняем соответствующие поля конкретными данными клиента придерживаясь типов этих полей (Смотрите сноску 1)
5. При нажатии на "Переход" открывается меню выбора дальнейшей работы:
6. При нажатии на "Визуальное отображение" происходит переход в новое окно "Визуальное отображение". В этом окне видны все записи с ранее заполненными полями с невыданными инструментами (Дата выдачи отсутствует).
В последнем столбце находится progressbar "Статус", отвечающий за время, остающееся до конца максимального срока на ремонт конкретного инструмента. Полоска зеленого цвета имеет рядом процент (от 100 до 0), отвечающий так же за последний срок выдачи.
Срок определяется по правилу: 40 дней от Даты принятия, если в БД стоит 0 в ячейке "Срочность" для данной записи - тогда checkbox в "Визуальном отображении" будет пустым; 15 дней, если стоит 1 - тогда в checkbox будет галочка
7. При выделении ячейки "Номер акта" для конкретной записи и последующего нажатия на "Информация" - "Детали" открывается таблица "Детали по текущему акту"
8. Чтобы добавить запись по детали (-ям), нужно нажать на кнопку "Добавить деталь" и ввести соответствующие данные по обслуживанию/ремонту инструмента так же придерживаясь типов этих полей (Смотрите сноску 2)
* Поле "Обшая цена" генерируется автоматически по формуле: Цена за шт (REAL) * Шт (REAL)
* Поле "ФИО" содержит выпадающий список работников, имеющихся в "Персонал" - "Управление работниками" в меню основной БД. Автоматически выбирается первая фамилия в списке. Чтобы поменять работника нужно нажать по этому выпадающему списку и выбрать верного работника.
* Чтобы заполнилось поле "ИТОГО", которое считается по формуле: Общая цена (REAL) + Сумма работы (REAL), нужно в верхнем меню выбрать "Подсчитать" - "Подсчет суммы" или использовать шорткат "Подсчет суммы". В поле появляется число
8.1 Чтобы удалить запись по детали, нужно нажать на ячейку "Название" для конкретной детали, которую нужно удалить и нажать на кнопку "Удалить деталь"

Поздравляю, Вы занесли всю информацию по первому инструменту! Сейчас будет информация по дополнительным функциям и работе с ними.

1. Работа с "Архив": При выделении ячейки "Номер акта" для конкретной записи и последующего нажатия на "Архив" показывается таблица "Детали для текущего акта" со всей информацией внтури нее.
Полезно, чтобы посмотреть какие детали были поставлены, на какую сумму осуществлен ремонт, кто поставлен в качетсве мастера.
Работает для всех записей, но подразумевается, что если не стоит "Дата выдачи", то всю информацию можно посмотреть в "Визуальном отображении" - "Детали по текущему акту" (п.5)
Если же "Дата выдачи" стоит, то в "Визуальном отображении" данная запись не попадет - тогда действительно используется "Архив" по назначению
2. Работа с "Генерация акта": При выделении ячейки "Номер акта" для конкретной записи и последующего нажатия на "Генерация акта" при условии что в папке C:\Test имеется неповрежденный не открытый файл Act.docx для текущего акта в этой же папке появится файл в формате "Act_%1", где %1 - номер акта (1 столбец)
В этом файле автоматически подставляются данные из записи по текущему акту из основной БД.
* Не изменять директорию/редактировать файл шаблона Act.docx для правильной работы программы!
3. Работа с "Экспорт": После нажатия на кнопку "Экспорт" в "Переход" - "Экспорт" открывается окно выбора файла (обзор директорий компьютера). Необходимо выбрать верную директорию куда необходимо сохранить файл в формате .xlsx, выбрать имя файла и нажать на кнопку "Сохранить".
После этого в указанной директории появляется Ваш файл со всеми данными из основной БД, включая заголовки.
* "Экспорт" полезен для бэкапов. Автоматически они не делаются на компьютере, поэтому систематически (по Вашему желанию) следует производить данную функцию, чтоб не потерять большое количество данных в непридвиденных ситуациях
4. Работа с "Управление работниками": После нажатия на кнопку "Управление работниками" в "Персонал" - "Управление работниками" открывается окно "Редактирование работников" с таблицей по текущим работникам внутри фирмы со столбцами все фамилии работников и их занимаемые должности (профессия).
4.1 Для начала необходимо добавить новую должность: для этого нажимаем на кнопку "Добавить должность", после чего открывается окно "Добавить должность", где в строке необходимо написать название должности и нажать на "ОК".
4.2 Затем необходимо добавить фамилию сотрудника: для этого нажимаем на кнопку "Добавить работника", после чего открывается окно "Добавить сотрудника", где в строке необходимо написать фамилию сотрудника и нажать на "ОК", после чего в выпадающем списке выбрать его должность (профессию).
Чтобы удалить работника или должность (на прошлых шагах) необходимо нажать на кнопку "Удалить работника" - удаляется информация по данному человеку или "Удалить должность" - удаляется конкретная должность
5. Работа с "Заработано": после нажатия на кнопку "Заработано" в "Персонал" - "Заработано" открывется окно с выбором фамилии (для кого будем смотреть статистику) в выпадающем списке. Выбираем фамилию.
Здесь указаны все инструменты, у которых в таблице "Детали по текущему акту" в столбце "ФИО" фамилия совпадает с выбранной ранее. Далее, если в основной БД указана "Дата выдачи" (п.9) нужно нажать на "З/П в графике" в верхнем меню и выбрать нужное Вам представление зарплаты для работника.
"Гистограмма": открывается окно со столбцами за все время, где по оси X представлена "Дата выдачи", по оси Y "Сумма работы" в виде суммы за весь этот день. При наведении мыши на интересующий стобец показывается "Сумма работы".
Для масштабирования (когда дат будет много) нужно воспользоваться колесиком мыши.
"Круговая диаграмма": открывается окно "Круговая диаграмма" с круговой диаграммой, раскрашенной в разные цвета (цвета выбираются случайно, возможны совпадения или слияния цветов).
Справа представлена краткая информация по визуализации в формате: Цвет (квадрат), Дата выдачи (dd.mm.yyyy): Сумма работы (REAL), которые взяты из таблицы "Заработано".
Так же слева изображен значок фильтра, который отвечает за выборку за период в виде выпадающего списка: Неделя, Месяц, Год. Автоматически при нажатии на "Круговая диаграмма" стоит выборка Неделя
6. Работа с "Установленные детали": После нажатия на кнопку "Детали" - "Установленные детали" открывается окно "Информация по всем деталям", куда автоматически поступает информация со всех таблиц "Детали по текущему акту". Полезная таблица для списания деталей в большом количестве актов.

Сноски:
1. Поля в БД
Номер акта INTEGER PRIMARY KEY AUTOINCREMENT,
ФИО TEXT, 
Адрес TEXT, 
Телефон INT, 
Название инструмента TEXT, 
Модель TEXT, 
Неисправность TEXT, 
Серийный номер TEXT,
Предоплата INT, 
Срочность BOOL,
Дата принятия DATE, 
Дата выдачи DATE
Где TEXT - любой текст длиной до 128 символов; INT - целое число от 0 до 2147483647; BOOL - булево значение, где 0 - несрочный ремонт (40 дней), 1 - срочный ремонт (15 дней); DATE - дата в формате dd.mm.yyyy

2. Поля в "Детали"
Название TEXT NOT NULL, 
Артикул TEXT NOT NULL, 
Шт INT NOT NULL, 
Цена за шт REAL NOT NULL, 
Общая цена REAL NOT NULL, 
ФИО TEXT, 
Сумма работы REAL, 
ИТОГО REAL
Где TEXT - любой текст длиной до 128 символов; INT - целое число от 0 до 2147483647; REAL - значения с плавающей точкой от 0 до 1.7976931348623157e308; NOT NULL - ограничение поля что оно не может быть нулевым (пустым)
